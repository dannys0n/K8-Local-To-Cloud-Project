# Kubernetes Development Environment Setup

## Overview
This repository provides a complete Kubernetes development environment setup with monitoring, databases, and management tools. The environment includes:
- **Monitoring**: Grafana and Prometheus
- **Databases**: Redis and PostgreSQL
- **Management**: Portainer UI
- **Testing**: HTTP echo service

## Setup Instructions

### Prerequisites
- Kubernetes cluster (minikube, kind, or cloud provider)
- `kubectl` configured
- Helm 3 installed

### Deployment
```bash
# Deploy all components
make up

# Deploy specific components
make monitors
make managers
make databases
make port-forward
```

### Services Access
| Service | URL | Credentials |
|---------|-----|-------------|
| Grafana | http://localhost:3000 | Admin password from monitoring secret |
| Prometheus | http://localhost:9090 | - |
| Portainer | http://localhost:9000 | Create admin on first visit |
| Redis | localhost:6379 | - |
| PostgreSQL | localhost:5432 | `postgres/postgres` |

### Database Credentials
- **Redis**: No credentials required
- **PostgreSQL**: 
  - User: `postgres`
  - Password: `postgres`
  - Database: `app`

### Cleanup
```bash
# Remove all components
make down

# Remove specific components
make cluster
make monitors
make managers
make databases
make port-forward
```

### Verification
```bash
# Check monitoring pods
kubectl get pods -n monitoring

# Test Redis connectivity
./src/scripts/ping-redis.sh

# Test PostgreSQL connectivity
./src/scripts/ping-postgres.sh
```

## Component Details

### Monitoring Stack
- **Prometheus**: Monitoring Kubernetes metrics
- **Grafana**: Visualization dashboard
- **Portainer**: Management UI for Kubernetes

### Database Stack
- **Redis**: In-memory data store
- **PostgreSQL**: Relational database with app-specific schema

### Management Tools
- **Portainer**: Kubernetes management UI
- **Helm**: Package manager for Kubernetes

## Configuration Files
- `src/databases/redis.yaml`: Redis deployment and service
- `src/databases/postgres.yaml`: PostgreSQL deployment and service
- `src/managing/portainer-values.yaml`: Portainer Helm chart configuration
- `src/scripts/port-forward.sh`: Local port forwarding configuration

## Notes
1. First-time Portainer setup requires creating an admin account
2. PostgreSQL database `app` is pre-created with `postgres` user
3. Redis and PostgreSQL use persistent storage
4. All services are exposed via local port forwarding for development use
